# nCounter 优化实施计划

## 概述
本文档记录 nCounter (WatchOS 拉力器/吊单杠追踪应用) 的优化方案和实施步骤。

---

## 1. 运动检测逻辑优化

### 1.1 当前问题
- **单维度检测**: 仅使用 Z 轴数据 (z > 0.2 / z < -0.8)
- **容易误触发**: 抬手看表、正常摆臂都会触发
- **阈值固定**: 不同用户手腕大小/佩戴位置不同，无法适应
- **缺乏状态管理**: 没有防抖机制，状态切换过于敏感

**当前代码位置**: `InterfaceController.swift:276-331`
```swift
func startRecord(z:Double, timer1:Timer) {
    if z > 0.2 {
        // 触发开始...
    }
}
```

### 1.2 优化方案

#### 1.2.1 多轴融合算法
**目标**: 使用 X+Y+Z 组合判断，提高检测准确性

**实现要点**:
- 计算三轴加速度的向量模: `magnitude = sqrt(x² + y² + z²)`
- 分析运动特征模式（吊单杠时的加速度特征）
- 设置动态阈值范围而非固定值

**技术细节**:
```
检测条件:
- 向量模稳定性: |magnitude - baseline| < threshold
- Z轴主导性: |z| > |x| && |z| > |y|
- 持续时间: 状态保持 > minimum_duration
```

#### 1.2.2 防抖动机制
**目标**: 避免瞬时抖动触发误判

**实现要点**:
- 添加状态持续时间检测（需保持稳定 1-2 秒才触发）
- 实现状态机：IDLE → DETECTING → CONFIRMED → ACTIVE
- 使用滑动窗口平均值过滤噪声

**状态机设计**:
```
IDLE (空闲)
  ↓ 检测到运动
DETECTING (检测中)
  ↓ 持续 1-2 秒稳定
CONFIRMED (确认)
  ↓ 开始计时
ACTIVE (运动中)
  ↓ 运动结束
IDLE
```

#### 1.2.3 自定义灵敏度设置
**目标**: 支持不同用户个性化配置

**实现要点**:
- 提供 3 档灵敏度：低/中/高
- 阈值范围：
  - 低灵敏度: z > 0.4, 稳定时间 2 秒
  - 中灵敏度: z > 0.2, 稳定时间 1.5 秒 (默认)
  - 高灵敏度: z > 0.1, 稳定时间 1 秒
- 设置持久化存储 (UserDefaults)

**UI 需求**:
- Watch App 设置页面添加灵敏度选择
- 显示当前灵敏度级别

#### 1.2.4 运动模式识别（未来方向）
**目标**: 使用机器学习识别"吊单杠"特征

**实现要点**:
- 收集训练数据：不同用户的吊单杠加速度数据
- 提取特征：频率、幅度、持续时间
- CoreML 模型集成
- 支持多种运动模式：吊单杠、引体向上、屈臂悬垂

**阶段规划**:
- Phase 1: 基于规则的检测（当前优化）
- Phase 2: 统计模型优化
- Phase 3: CoreML 机器学习模型

## 2. 计时和计数机制优化

### 2.1 当前问题
- **自动计数不合理**: 每 10 秒自动 +1，不是真实运动次数
- **缺乏实时反馈**: 只显示累计秒数，看不到每组时间
- **没有组间管理**: 无法记录多组训练
- **缺少目标设定**: 用户无法设置目标时长或次数

**当前代码位置**: `InterfaceController.swift:139-155`
```swift
func countMethod() {
    self.countSecondTimer = Timer.scheduledTimer(withTimeInterval: 1.0, repeats: true) { timer3 in
        self.pullUpAccumulateTime += 1
        if self.pullUpAccumulateTime % 10 == 0 {
            self.pullUpCount += 1  // 每10秒+1
        }
    }
}
```

### 2.2 优化方案

#### 2.2.1 实时运动检测计数
**目标**: 从时间计数改为真实的运动次数检测

**实现要点**:
- 检测引体向上的完整动作周期
- 一个完整周期：上去(z > threshold) + 下来(z < -threshold)
- 每完成一个周期，计数 +1
- 实时显示当前次数和累计时间

**动作检测逻辑**:
```
状态 A: 手臂伸直 (z < -0.8)
  ↓ 向上拉
状态 B: 下巴过杠 (z > 0.2)
  ↓ 向下放
状态 A: 手臂伸直 (z < -0.8)
  → 完成一次，计数 +1
```

#### 2.2.2 精确计时系统
**目标**: 分别记录每组的时间和休息时间

**实现要点**:
- 实时显示：
  - 当前组持续时间
  - 总累计时间
  - 当前组次数
  - 总次数
- 组间休息检测：
  - 检测到用户停止运动 > 30 秒
  - 自动进入休息模式
  - 显示休息时长

**数据结构**:
```swift
struct WorkoutSession {
    var sets: [WorkoutSet]
    var totalDuration: TimeInterval
    var totalReps: Int
    var startTime: Date
}

struct WorkoutSet {
    var reps: Int
    var duration: TimeInterval
    var startTime: Date
    var restDuration: TimeInterval?
}
```

#### 2.2.3 组间休息管理
**目标**: 智能管理多组训练

**实现要点**:
- 自动检测休息开始（运动停止 > 30 秒）
- 显示休息倒计时（建议休息 60-90 秒）
- 休息结束提醒（震动提示）
- 记录每组数据：
  - 第几组
  - 每组次数
  - 每组时间
  - 组间休息时长

**UI 展示**:
```
运动中:
┌─────────────────┐
│   第 3 组       │
│   次数: 8       │
│   时间: 00:45   │
│   总计: 22 次   │
└─────────────────┘

休息中:
┌─────────────────┐
│   休息中        │
│   00:35 / 01:00 │
│   [跳过休息]    │
└─────────────────┘
```

#### 2.2.4 目标设定系统
**目标**: 允许用户设置训练目标

**实现要点**:
- 支持目标类型：
  - 次数目标（如：50 个引体向上）
  - 时间目标（如：坚持 5 分钟）
  - 组数目标（如：5 组，每组 10 个）
- 实时进度显示：
  - 进度条/百分比
  - 剩余次数/时间
- 达成提醒：
  - 目标完成时震动 + 音效
  - 显示庆祝动画

**数据模型**:
```swift
enum GoalType {
    case totalReps(Int)        // 总次数
    case totalDuration(Int)    // 总时长(秒)
    case sets(repsPerSet: Int, numberOfSets: Int)  // 组数×每组次数
}

struct WorkoutGoal {
    var type: GoalType
    var progress: Double  // 0.0 - 1.0
    var isCompleted: Bool
}
```

#### 2.2.5 成就系统
**目标**: 增加训练趣味性和持续性

**实现要点**:
- 成就类型：
  - 🏅 首次完成 10 个
  - 🥈 连续 3 天训练
  - 🥇 单次 50 个
  - 💪 累计 1000 个
- 成就展示：
  - 训练结束时显示获得成就
  - 查看历史成就列表
- 数据持久化：
  - 记录总训练次数
  - 记录总训练时长
  - 记录最佳成绩

## 3. 其他重要优化

### 3.1 性能优化
**当前问题**:
- 加速度采样率 60Hz 过高，耗电严重

**优化方案**:
- 降低采样率到 30Hz
- 使用 DeviceMotion 替代原始加速度
- 添加低功耗模式

**代码位置**: `InterfaceController.swift:162-196`

### 3.2 代码质量
**当前问题**:
- 两个 CMMotionManager 实例 (motionManager + motion)
- 大量注释代码
- Timer 内存泄漏风险
- API Key 硬编码

**优化方案**:
1. 统一 CMMotionManager 实例
2. 删除注释代码，使用 Git 管理历史
3. Timer 使用 weak self 避免循环引用
4. API Key 移到配置文件

**清理位置**:
- Lines 42-79: 注释代码
- Lines 122-134: 注释代码
- Lines 316-330: 注释代码
- Lines 401-448: 注释代码

### 3.3 架构优化
**当前问题**:
- 所有逻辑都在 InterfaceController (505 行)
- 缺少分层设计

**优化方案**:
```
架构分层:
├── Controllers/
│   └── InterfaceController.swift  (仅 UI 逻辑)
├── Services/
│   ├── MotionDetectionService.swift
│   ├── TimerService.swift
│   └── NetworkService.swift
├── Models/
│   ├── WorkoutSession.swift
│   ├── WorkoutGoal.swift
│   └── Achievement.swift
└── Helpers/
    ├── StateMachine.swift
    └── DataPersistence.swift
```
